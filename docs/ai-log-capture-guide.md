# Как снять лог по рикошетам ИИ (пошагово)

Ниже — короткая инструкция без кода игры, только действия в браузере.

## 1) Запустите матч

Сделайте обычный запуск игры и доведите до ситуации, где:
- между ИИ и целью есть кирпичи,
- ИИ должен был бы использовать рикошет,
- но выбирает прямой удар или запасной сценарий.

Важно: сначала создайте 2–5 таких ходов подряд, чтобы в логе была серия, а не один случай.

## 2) Откройте консоль браузера

- Chrome/Edge: `F12` → вкладка **Console**.
- Safari/Firefox: откройте инструменты разработчика и вкладку консоли.

## 3) Выведите короткий список логов

Вставьте в консоль:

```js
AI_PRINT_PATH_PASSPORT_LOGS(20)
```

Это покажет последние 20 записей по этапам выбора маршрутов.

## 4) Выгрузите полный JSON

Вставьте в консоль:

```js
AI_EXPORT_PATH_PASSPORT_LOGS(20)
```

Команда вернёт JSON-текст. Скопируйте его целиком и сохраните в файл, например:

`ai-path-passport-log-2026-04-21.json`

## 5) Что прислать для анализа

Чтобы быстро найти корневую причину, достаточно прислать:

1. JSON из `AI_EXPORT_PATH_PASSPORT_LOGS(20)`.
2. Краткое описание карты и ситуации (кто стрелял, где были кирпичи).
3. Что ожидалось и что произошло фактически.

## Как понять результат «на глаз»

- `finalStatus: "selected"` — ИИ выбрал кандидат.
- `finalStatus: "rejected"` — все кандидаты отвалились.
- `killedAtStage` — на каком этапе умер кандидат.
- `killedByReason` — почему именно отвалился.

Это и есть ключ к ответу на вопрос: «ИИ вообще видит рикошет или теряет его на проверках».
